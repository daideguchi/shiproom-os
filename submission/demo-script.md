# Demo Script — Shiproom OS

Target length: 2-3 minutes

## 0:00 - 0:20

Show the live app.

Narration:

```text
AI makes it easy to start. Shiproom OS is about finishing. It helps a solo builder turn a rough product idea into a launch packet that can actually be reviewed and shipped.
```

## 0:20 - 0:55

Fill or show the product intake.

Narration:

```text
The builder starts with a rough idea, the target user, the core pain, and a timebox. This keeps the product grounded before AI-generated scope turns into noise.
```

## 0:55 - 1:30

Click Build Packet and show Product Frame, Scope, Proof Checklist, and Launch Copy.

Narration:

```text
Shiproom OS creates the product frame, cuts the first version down, writes a proof checklist, and drafts launch copy. The important part is that it does not say the product is done just because text exists.
```

## 1:30 - 2:05

Show Evidence Ledger and Novus status.

Narration:

```text
The evidence ledger is the difference between a demo and a product. Public URL, user signal, demo video, build journey, human review, and Novus proof all have visible states. If Novus is not installed, the tool says so.
```

## 2:05 - 2:35

Show Markdown submission brief and saved snapshots.

Narration:

```text
The launch packet can be exported as JSON or Markdown, and snapshots can be saved locally while the builder compares directions. The result is a small operating room for getting from idea to public launch.
```

## 2:35 - 2:50

Close on the product thesis.

Narration:

```text
Everyone can generate. Fewer people ship. Shiproom OS closes that last mile.
```
